build_request_uri (uri, params, container)

Testplan URIs can be relative or absolute. Either one can contain variables in the form :varname, as in: 

http://example.com/product/:productid

This method creates a new URI object using the @base_uri if necessary and replaces all variables with literal values. If no replacement value can be found, the variable will remain.

find_replacement_value (name, params, container, base_uri)

Testplan URIs can contain variables in the form :varname. This method looks at the request parameters and then at the usecase’s resource hash for a replacement value. If not found, returns nil.

[show source] 
     # File lib/stella/client.rb, line 293
293:     def find_replacement_value(name, params, container, base_uri)
294:       value = nil
295:       #Stella.ld "REPLACE: #{name}"
296:       #Stella.ld "PARAMS: #{params.inspect}"
297:       #Stella.ld "IVARS: #{container.instance_variables}"
298:       if name.to_sym == :HOSTNAME && !base_uri.nil?
299:         value = base_uri.host 
300:       elsif params.has_key?(name.to_sym)
301:         value = params.delete name.to_sym
302:       elsif container.resource?( name)
303:         value = container.resource name
304:       elsif Stella::Testplan.global?(name)
305:         Stella::Testplan.global(name)
306:       end
307:       value
308:     end
find_response_handler (container, req)

Find the appropriate response handler by executing the HTTP response status against the configured handlers. If several match, the first one is returned.
